测试不同的文档变体，以找到实际让代理在压力下发现和使用技能的内容。


## 测试协议

对于每个变体：

1. **首先运行 NULL 基线**（无技能文档）
   - 记录代理选择哪个选项
   - 捕获确切的合理化

2. **用相同场景运行变体**
   - 代理是否检查技能？
   - 代理是否在找到时使用技能？
   - 如果违规，捕获合理化

3. **压力测试** - 添加时间/沉没成本/权威性
   - 代理在压力下仍然检查吗？
   - 记录合规何时崩溃

4. **元测试** - 问代理如何改进文档
   - "你有文档但没有检查。为什么？"
   - "文档如何能更清晰？"

## 成功标准

**变体成功如果：**
- 代理未经提示检查技能
- 代理在行动前完全阅读技能
- 代理在压力下遵循技能指导
- 代理无法合理化掉合规

**变体失败如果：**
- 代理即使没有压力也跳过检查
- 代理在没有阅读的情况下 "适应概念"
- 代理在压力下合理化掉
- 代理将技能视为参考而非要求

## 预期结果

**NULL：** 代理选择最快路径，无技能意识

**变体 A：** 代理可能在没有压力时检查，在压力下跳过

**变体 B：** 代理有时检查，容易合理化掉

**变体 C：** 强合规但可能感觉太僵化

**变体 D：** 平衡，但较长 - 代理会内化它吗？
