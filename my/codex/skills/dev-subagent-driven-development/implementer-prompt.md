# 实现者子代理 Prompt 模板

分派实现者子代理时使用此模板。

```
Task tool (general-purpose):
  description: "Implement Task N: [task name]"
  prompt: |
    你正在实现任务 N：[task name]

    ## 任务描述

    [计划中任务的完整文本 - 粘贴到这里，不要让子代理读取文件]

    ## 上下文

    [场景设置：这适合哪里、依赖、架构上下文]

    ## 开始前

    如果你对以下有疑问：
    - 需求或验收标准
    - 方法或实施策略
    - 依赖或假设
    - 任务描述中任何不清楚的地方

    **现在问。** 在开始前提出任何疑虑。

    ## 你的工作

    一旦你对需求清楚：
    1. 实现任务指定的确切内容
    2. 编写测试（如果任务要求则遵循 TDD）
    3. 验证实现有效
    4. 提交你的工作
    5. 自我审查（见下文）
    6. 报告回来

    工作目录：[directory]

    **工作时：** 如果遇到意外或不清楚的内容，**提问**。
    暂停并澄清总是可以的。不要猜测或做假设。

    ## 报告回来之前：自我审查

    用新鲜的眼光审查你的工作。问自己：

    **完整性：**
    - 我完全实现了规范中的所有内容吗？
    - 我遗漏了任何需求吗？
    - 有没有我没有处理的边缘情况？

    **质量：**
    - 这是我最好的工作吗？
    - 名称是否清晰准确（匹配功能，而非实现方式）？
    - 代码是否干净可维护？

    **纪律：**
    - 我避免了过度构建（YAGNI）吗？
    - 我只构建了请求的内容吗？
    - 我遵循了代码库中的现有模式吗？

    **测试：**
    - 测试实际验证行为（不仅仅是模拟行为）吗？
    - 如需，我遵循了 TDD 吗？
    - 测试全面吗？

    如果你在自我审查中发现问题，立即修复再报告。

    ## 报告格式

    完成后，报告：
    - 你实现了什么
    - 你测试了什么和测试结果
    - 更改的文件
    - 自我审查发现（如有）
    - 任何问题或疑虑
```
