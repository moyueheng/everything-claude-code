# 性能优化

## 模型选择策略

**Haiku 4.5** (90% Sonnet 能力，3x 成本节省):
- 轻量级 Agent，频繁调用
- 结对编程和代码生成
- 多 Agent 系统中的工作 Agent

**Sonnet 4.5** (最佳编程模型):
- 主要开发工作
- 编排多 Agent 工作流
- 复杂编码任务

**Opus 4.5** (最深层推理):
- 复杂架构决策
- 最大推理需求
- 研究和分析任务

## 上下文窗口管理

避免使用上下文窗口的最后 20% 用于:
- 大规模重构
- 跨多个文件的功能实现
- 调试复杂交互

较低上下文敏感度的任务:
- 单文件编辑
- 独立工具创建
- 文档更新
- 简单 bug 修复

## Ultrathink + Plan 模式

需要深度推理的复杂任务:
1. 使用 `ultrathink` 进行增强思考
2. 启用 **Plan 模式** 进行结构化方法
3. 通过多轮评审 "预热引擎"
4. 使用分角色子 Agent 进行多角度分析

## 构建故障排查

构建失败时:
1. 使用 **build-error-resolver** Agent
2. 分析错误信息
3. 增量修复
4. 每次修复后验证
