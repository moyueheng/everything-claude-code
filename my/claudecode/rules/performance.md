# 性能优化

## 上下文窗口管理

避免使用上下文窗口的最后 20% 用于:
- 大规模重构
- 跨多个文件的功能实现
- 调试复杂交互

较低上下文敏感度的任务:
- 单文件编辑
- 独立工具创建
- 文档更新
- 简单 bug 修复

## Ultrathink + Plan 模式

需要深度推理的复杂任务:
1. 使用 `ultrathink` 进行增强思考
2. 启用 **Plan 模式** 进行结构化方法
3. 通过多轮评审 "预热引擎"
4. 使用分角色子 Agent 进行多角度分析

## 构建故障排查

构建失败时:
1. 使用 **build-error-resolver** Agent
2. 分析错误信息
3. 增量修复
4. 每次修复后验证
